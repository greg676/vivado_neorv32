## Key Differences vs. Canonical (neorv_ddr build)

The canonical doc (§2) describes the `neorv_ddr` block design with DDR3 MIG + AXI DMA + AXI FIFOs. **This is not what was built today.**

What changed in `neorv_noddr`:
1. **No DDR3 MIG** — no external memory at all
2. **No AXI interconnect** — no smartconnect
3. **No DMA** — IO_DMA_EN=true but XBUS_EN=false → DMA has no bus path, effectively disabled
4. **SLINK enabled** — AXI-Stream TDM audio data path (sd_in_codec_master_s_0 → SLINK RX)
5. **XBUS_EN=false** — this is CRITICAL: XBUS is the bridge from NeoRV32 to external Xilinx AXI IP. Without it, the DMA/SPI/TWI/UART are all internal-only
6. **Audio path changed**: `sd_in_codec_master_s_0` (AXI-Stream master) captures TDM data from CS42448 — connects to SLINK's RX stream interface
7. **ISA upgraded**: `M` (multiply) + `C` (compressed) added; IMEM/DMEM both 128KB (was 32KB/8KB)

## What This Actually Gives You

With XBUS_EN=false, this is a **bare NeoRV32 with internal peripherals only**:
- ✅ JTAG debug (OCD) works
- ✅ UART0 works (but no USB-UART pins in this top-level wrapper? need to check)
- ✅ TWI works for I²C scanning
- ✅ SPI works
- ✅ GPIO works
- ❌ DMA has no path to external memory (XBUS disabled)
- ❌ No DDR3
- ❌ No FT601
- ✅ SLINK can receive TDM audio from sd_in_codec_master_s_0

The SLINK RX FIFO (32KB) is the only place TDM data can land — it's the audio buffer. Firmware reads from SLINK_DATA register.
