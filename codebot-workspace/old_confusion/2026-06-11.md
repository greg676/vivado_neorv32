# Session Notes — 2026-06-11

## NeoRV32 Debug Console — Major Breakthrough Session

### Critical Bug Fixes Discovered
1. **UART RX bug (root cause of "no keystrokes")**: `neorv32_uart0_char_received()` only checks the RX_NEMPTY flag — it does NOT read/consume the byte. Must use `neorv32_uart0_char_received_get()` to actually read data from the UART FIFO. This was why the terminal appeared to accept no input — the same byte stayed in the FIFO forever.
2. **UART baud rate**: Confirmed 19200 baud at PRSC=3, BAUD=130 gives actual 19231 Hz (0.16% error). Formula: `baud = f_main / (4^prsc × 2 × (baud_div+1))`. NOT the same as TWI formula.
3. **I2C read bug**: In repeated-start read, the address byte must use ACK (0), not NACK (1). Only the final data byte gets NACK (1). Using NACK on the address phase caused every other register read to return 0x91 (the address byte itself).
4. **I2C speed formula**: `f_SCL = f_main / (4 × prescaler × (1+CDIV))`. CDIV is only 4 bits (0-15). Correct config for ~100 kHz at 160 MHz: PRSC=3 (prescaler=64), CDIV=5 → 104,167 Hz.
5. **hex printf bug**: `neorv32_uart0_printf` does NOT support `%02x`, `%08X`, or width specifiers. Use helper functions `phex(byte)` and `phex32(uint32_t)` instead.

### GPIO Pin Mapping (confirmed from XDC)
| GPIO bit | Pin | Name | Function |
|----------|------|------|----------|
| 0 | E13 | gpio0 | General purpose (NOT SPI CS) |
| 1 | E14 | gpio1 | General purpose |
| 2 | D14 | RSTCN | Codec reset (HIGH=active) |
| 3 | T5 | RSTDN | Display reset (HIGH=active) |
| 4 | U5 | gpio4 | General purpose |
| 5 | — | CLK_EN | clk_wiz_1 reset (HIGH=running) |

⚠️ Dout_0[0]=V9 is MCLK output (shares pin with nothing — it IS the SPI CS0 pin in some designs)
⚠️ Dout_0[1]=V8 and Dout_0[2]=U6 overlap with SPI CS1/CS2 pins
