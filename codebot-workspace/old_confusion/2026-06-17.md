- ISA: RV32I + Zicond + M + Zba/Zbb/Zbkb/Zbkx/Zihpm
- Clock: 160 MHz
- IMEM: 128 KB @ 0x00000000
- DMEM: 128 KB @ 0x80000000
- BOOTROM: ENABLED (boot mode 0)
- OCD: ENABLED
- UART0: ENABLED (RX FIFO 16, TX FIFO 16)
- TWI: ENABLED (FIFO 8)
- SPI: ENABLED (FIFO 256)
- GPIO: ENABLED (1 input, 6 outputs) ÔÇö but **IO_GPIO_DIR_EN=false**, **IO_CLINT_EN=false**

## CS42448 I┬▓C ÔÇö BREAKTHROUGH

### The Wrong Path (my J-Link direct reg pokes)
- TWI state machine needs continuous clock cycles from RUNNING CPU
- When CPU halted via J-Link, START/STOP/data state transitions stall
- Direct register writes to TWI control register (0xFFF90000) gave all NACKs

### The Right Path (debug_console firmware)
- Use pre-built `debug_console` firmware loaded via J-Link at IMEM
- UART interface provides `i2c scan` and `i2c write8` / `i2c read8` commands
- TWI setup: `neorv32_twi_setup(6, 15, 0)` = prsc=6 (clk/2048), cdiv=15, no clock stretching
- f_twi Ôëł 1.22 kHz (slow but reliable)

### CS42448 Register Read Results
| Reg | Value | Meaning |
|-----|-------|---------|
| 0x01 | 0x04 | Chip ID = CS42448 Ôťů |
| 0x02 | 0x00 | PDN clear, codec powered up Ôťů |
| 0x03 | 0xF0 | Interface Control 1 (correct) |
| 0x04 | **0x01** | **DIF = Left-Justified ÔÇö needs 0x36 for TDM** |
| 0x05 | 0x00 | ADC Control |
| 0x06 | 0x00 | DAC Control |
| 0x07 | 0xE4 | Misc Control |

## Next: Write TDM Config

## KEY LESSONS (hard-won)
1. **J-Link direct peripheral pokes don't work for TWI** ÔÇö CPU must be running
