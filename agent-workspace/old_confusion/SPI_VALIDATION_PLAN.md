[0, "# SPI Bus Validation Plan\n\nValidate wiring and basic communication with SPI peripherals:\n- Display (ILI9341) on CS0\n- Touch (XPT2046) on CS1\n- SD card (SDHC) on CS2\n- Flash (W25Q64) on CS3 (SPI-only access)\n\n## 🔧 Prerequisites\n- FPGA programmed with a bitstream that has NeoRV32 SPI controller enabled\n- J-Link connected to NeoRV32 OCD (independent JTAG chain)\n- Toolchain: `riscv-none-elf-gcc` working\n- Console: RP2040 USB-UART bridge at 19200 baud\n\n## 📋 Firmware to Build\nCreate `spi_test` firmware that:\n1. Initializes NeoRV32 SPI controller (master mode)\n2. Provides UART menu to select device and run tests\n3. Implements low-level SPI tx/rx functions\n4. Includes device-specific probe sequences\n\n### SPI Controller Setup\n```c\n// SPI0 configuration (peripherals.h)\ndefine SPI_PRSC   3   // clk/8 = 20 MHz (max for NeoRV32 SPI)\ndefine SPI_FRM    0   // standard frame\n\define SPI_CPOL  0   // clock polarity low\n\define SPI_CPHA  0   // clock phase sample on leading edge\n\neorv32_spi_setup(SPI_PRSC, SPI_FRM, SPI_CPOL, SPI_CPHA);\n```\n\n### Peripheral Chip Selects (GPIO)\n```c\n// GPIO pins for SPI chip select (active low)\ndefine SPI_CS_DISPLAY  (1UL << 0)  // GPIO[0] = A15\ndefine SPI_CS_TOUCH    (1UL << 1)  // GPIO[1] = A16\ndefine SPI_CS_SD       (1UL << 2)  // GPIO[2] = E16\ndefine SPI_CS_FLASH    (1UL << 3)  // Assume GPIO[3] if available, else reassign\n\n// Deselect all\n\neorv32_gpio_port_set(0x00000000);\n\n// Select device (active low)\n\neorv32_gpio_port_clr(SPI_CS_DEVICE);\n\n// Deselect device\n\neorv32_gpio_port_set(SPI_CS_DEVICE);\n```\n\n## 🧪 Test Procedures\n\n### 1. Display (ILI9341) - CS0\n**Expected**: LCD controller responds to register reads\n\n**Test sequence**:\n1. Assert CS0 (GPIO[0] low)\n2. Send: 0x00 (NOP) - should have no effect\n3. Send: 0x09 (Read Display Power Mode) + read 1 byte\n4. Send: 0x0A (Read Display MADCTL) + read 1 byte\n5. Send: 0x0B (Read Display Pixel Format) + read 1 byte\n6. Deassert CS0\n\n**Success**: Returns sensible values (e.g., 0x00 for reset state, 0x08 for 16-bit RGB565)\n\n### 2. Touch (XPT2046) - CS1\n**Expected**: Touch controller responds with CHIP_ID = 0x09\n\n**Test sequence**:\n1. Assert CS1 (GPIO[1] low)\n2. Send: 0x00 (CHIP_ID register read) + read 1 byte\n3. Deassert CS1\n\n**Success**: Returns 0x09 (XPT2046/ADS7843 identifier)\n\n### 3. SD Card - CS2\n**Expected**: Responds to CMD0 with R1 = 0x01 (idle state)\n\n**Test sequence**:\n1. Assert CS2 (GPIO[2] low)\n2. Send 10× 0xFF (clock pulses to wake up card)\n3. Send CMD0 (0x40 | 0x00) + arg[31:0] = 0 + CRC = 0x95\n4. Read response (should be R1 = 0x01)\n5. Send CMD8 (if SDHC) to check voltage\n6. Deassert CS2\n\n**Success**: CMD0 returns R1 = 0x01 (card in idle state)\n\n### 4. Flash (W25Q64) - CS3\n**Expected**: Responds to RDID with Manufacturer ID = 0xEF (Winbond)\n\n**Test sequence**:\n1. Assert CS3 (GPIO[3] low or assigned pin)\n2. Send: 0x9F (RDID command) + read 3 bytes\n3. Deassert CS3\n\n**Success**: Returns [0xEF, 0x40, 0x17] (Winbond W25Q64 64Mb)\n\n## 📍 Pin Map Verification\nCross-check against LOCKED pin map:\n\n| Signal | FPGA Pin | Function |\n|--------|----------|----------|\n| SPI0_SCLK | (from clk_wiz or internal) | SPI Clock |\n| SPI0_MOSI | (NEORV32 SPI0_MOSI) | Master Out Slave In |\n| SPI0_MISO | (NEORV32 SPI0_MISO) | Master In Slave Out |\n| SPI_CS0   | **GPIO[0] = A15** | ILI9341 Display CS |\n| SPI_CS1   | **GPIO[1] = A16** | XPT2046 Touch CS |\n| SPI_CS2   | **GPIO[2] = E16** | SD Card CS |\n| SPI_CS3   | **TBD** | W25Q64 Flash CS (if GPIO[3] available) |\n\n## 🛠️ Build & Load\n```bash\n# Build\nmake clean all BIN=spi_test\n\n# Load via J-Link GDB\nJLinkGDBServer -device RISC-V -if JTAG -speed 20000 -port 2331\n# In another terminal:\nriscv-none-elf-gdb build/spi_test.elf\n(gdb) target remote :2331\n(gdb) load\n(gdb) mon reset\n(gdb) continue\n\n# Connect to UART console\npython3 scripts/rp2040_uart_bridge.py\n# → spi_test menu appears\n```\n\n## 📌 Success Criteria\n- At least one SPI device responds correctly to its probe sequence\n- No bus lockups (ability to deselect and retry)\n- Chip select mapping matches pin map\n\n## 📄 Output\nAfter validation, update:\n- `WORKLOG.md` with test results\n- Critic knowledge: Add SPI validation notes to `codecbot-critic-project` collection\n- Share findings with Claw for any wiring corrections needed\n\n*Use this plan to validate SPI before moving to firmware development.*"]
