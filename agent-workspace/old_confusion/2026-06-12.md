## NeoRV32 I2C Read — CRITICAL BUG DISCOVERY

### The Problem
CS42448 register reads return alternating garbage (0x00/0xFF or 0x03/0x00 pattern) for registers 0x08+. Registers 0x01-0x07 sometimes read correctly. Results vary between runs.

### Investigation Path
1. **v14-v19 firmware iterations**: Added per-register write+verify, RX FIFO drains, `neorv32_twi_setup` before reads, delays up to 10ms, slowed I2C to 1.2kHz — all showed same alternating garbage
2. **Manual `i2c read` commands**: Also show alternating pattern for 0x08+ registers
3. **v20 minimal test**: Used EXACT official NeoRV32 `read_i2c_byte()` function verbatim — still shows garbage, different values each run
4. **TWI_CTRL register**: FIFO depth = 1 entry (bits 15:12 = 0x0). Not the root cause since each transfer reads immediately.

### Root Cause Hypothesis
**Missing I2C pull-up resistors.** The Alchitry Pt V2 + Br breakout may not have external pull-ups on SCL (B20) and SDA (C14). Without pull-ups:
- Bus floats when not driven
- ACK/NACK sampling unreliable
- Data reads return random values
- Behavior varies between runs (capacitive coupling)

The official NeoRV32 I2C probe example works on the Arty board (which has proper I2C pull-ups) but fails on the Alchitry Pt V2.

### Evidence
- Writes sometimes work (ACK detected correctly) but reads fail
- Alternating 0x00/0xFF pattern suggests bus is floating to random states
- Different results each run (v20 showed 0x03/0x00 then 0x00/0x14/0x21)
- ERR on some reads indicates NACK detection

### Fix Options
1. **Enable FPGA internal pull-ups** on B20 and C14 in XDC (add `PULLUP` constraint)
2. **Add external pull-up resistors** (2.2k-4.7k) on Br breakout SCL/SDA lines
3. **Check if Br breakout already has pull-ups** — might need different pins or jumper configuration

### What DOES Work
- I2C writes succeed (ACK detected) — codec init config registers written successfully
- Chip ID read (0x01=0x04) works consistently
- Manual `i2c write` commands work
- UART passthrough via RP2040 bare-metal .uf2 works
- JTAG loading and debugging works
